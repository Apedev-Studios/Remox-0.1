Chapter I

Language Architecture

Remox v0.8 is not interpreted line-by-line. It is a real, ahead-of-time compiled language — source code passes through four distinct phases before a single statement executes.

The Compilation Pipeline
Source Code
.remox file
→
Lexer
Tokens
→
Parser
AST
→
Compiler
Validation + Folding
→
Runtime
Execution

The Lexer reads raw source text character by character, producing a flat stream of Tokens — keywords, literals, operators, identifiers. It handles string escape sequences, floating-point numbers, scientific notation, and two comment styles (// and ##).

Remox
// Variables aur basic syntax
let name = "Remox"
let version = 0.8
let ready = true

## String interpolation
say "Hello from {name} v{version}!"

// Scientific notation supported
let big = 1.5e10
let small = 2.5e-3
Parser & Abstract Syntax Tree

The Parser consumes tokens using recursive descent and builds an AST (Abstract Syntax Tree). It handles operator precedence correctly via a layered call chain:

parse_expr
→
parse_pipe
|>
→
parse_ternary
? :
→
parse_or / and
→
parse_equality
== != is
→
parse_comparison
< > <= >=
→
parse_add/mul
+ - * / %
→
parse_postfix
. [] ()
→
parse_primary
literals, idents

This ensures a + b * c is always parsed as a + (b * c), not (a + b) * c. The pipe operator |> sits at the top of the chain with the lowest precedence, allowing elegant data pipelines.

The Ahead-of-Time Compiler (v0.8)

The v0.8 upgrade introduced a real compile phase that runs entirely before execution. This enables forward references — calling a function defined later in the file — which pure interpreters cannot do.

⚙️
Two-Pass Compilation

Pass 1 (Hoisting): All fn, struct, impl, trait declarations are registered regardless of their position.
Pass 2 (Validation + Folding): Constant arithmetic is folded at compile time, struct references are validated, warnings are collected.

Remox — Constant Folding Example
// Ye expression compile-time pe fold ho jaata hai
// Runtime pe koi calculation nahi hoti
let area = 10 * 20          // → IntLit(200)
let pi_sq = 3.14 * 3.14     // → FloatLit(9.8596)
let flag = true and false   // → BoolLit(false)

// Forward reference — function baad mein defined hai, pehle call kar sakte ho
say greet("Aahil")

fn greet(name) {
    "Hello, {name}!"
}
Chapter II

Core Syntax

Remox is built to be the simplest language you've ever used. Every construct has a clean, readable keyword — no curly-brace ceremonies, no semicolons required, no magic punctuation.

Variables & Data Types
let
Variable Declaration
Declare one or multiple variables in one line.
let x = 42 let a, b = 10, 20 let name = "Remox"
Int / Float
Numeric Types
64-bit integers and 64-bit floats, auto-detected from literals.
let n = 100 // Int let f = 3.14 // Float let s = 1.5e10 // Scientific
List / Map
Collections
Dynamic ordered lists and key-value maps with rich built-in methods.
let nums = [1, 2, 3, 4] let user = {name: "Dev", age: 21} say nums.sort() say nums.unique()
Range
Range Type
Elegant range syntax for iteration and slicing.
let r = 1..10 each n in 1..5 { say n }
null
Null Safety
Explicit null values with safe indexing — null never crashes, it returns null.
let x = null let val = list?[99] // null if out of bounds
ok / err
Result Type
Explicit success and error values for safe error handling without exceptions.
let r = ok(42) let e = err("failed") say r.is_ok() // true say r.unwrap() // 42
String Interpolation
Remox
let name = "Aahil"
let score = 99

// {variable} andar directly embed kar sakte ho
say "Player {name} scored {score} points!"

// Escape sequences bhi kaam karte hain
let msg = "Line 1\nLine 2\tTabbed"
Control Flow
Remox — All Control Flow Constructs
// ── IF / ELSE ────────────────────────────────────────────
if score >= 90 {
    say "Distinction!"
} else {
    say "Keep trying"
}

// ── WHEN (pattern matching on value) ─────────────────────
when grade {
    "A" => say "Excellent"
    "B" => say "Good"
    else => say "Try again"
}

// ── MATCH (expression-style, returns value) ───────────────
let label = match version {
    0.8 => "Beta"
    1.0 => "Stable"
    _   => "Unknown"
}

// ── LOOP (fixed count) ────────────────────────────────────
loop 5 {
    say "Counting..."
}

// ── EACH (iterator) ──────────────────────────────────────
each item in [1, 2, 3] {
    say item * 2
}

// ── TERNARY ──────────────────────────────────────────────
let result = score >= 50 ? "Pass" : "Fail"

// ── LIST COMPREHENSION ────────────────────────────────────
let evens = [x for x in 1..20 if x % 2 == 0]
Functions
Remox — Function Variations
// ── Basic function ────────────────────────────────────────
fn square(n) {
    n * n
}
say square(9)   // 81

// ── Named parameters (Feature 21) ────────────────────────
fn greet(name:, city:) {
    "Hello {name} from {city}!"
}
greet(name: "Aahil", city: "Apeion")

// ── Default parameters (Feature 22) ──────────────────────
fn power(base, exp: 2) {
    base * base  // simplified
}
power(5)        // uses exp=2
power(5, 3)     // uses exp=3

// ── Pipe operator  |> (Feature 26) ───────────────────────
let result = [3, 1, 4, 1, 5] |> sort |> unique

// ── Explicit return ───────────────────────────────────────
fn check(x) {
    if x < 0 { return "negative" }
    "positive"
}

// ── Spread operator (Feature 23) ──────────────────────────
let a = [1, 2, 3]
let b = [...a, 4, 5]   // [1, 2, 3, 4, 5]
💡
Last Expression Rule

In Remox, the last expression in a function body is automatically the return value — just like Rust. You don't need an explicit return unless you're returning early.

Chapter III

Advanced Syntax

Beyond the basics — Remox supports full OOP via Structs, Impl, and Traits; functional programming via Lambdas and Closures; and concurrent programming via Async/Await.

Structs, Impl & Traits
Remox — OOP System
// ── Struct declaration (Feature 27) ──────────────────────
struct Player {
    name, score, level
}

// ── Auto-constructor (Feature 28) ─────────────────────────
let p = Player(name: "Aahil", score: 0, level: 1)

// ── Field access ──────────────────────────────────────────
say p.name      // "Aahil"
p.score = 100   // Field assignment

// ── Impl block — methods (Feature 29) ────────────────────
impl Player {
    fn greet(self) {
        say "I am {self.name}, Level {self.level}"
    }

    fn level_up(self) {
        self.level = self.level + 1
    }
}

p.greet()
p.level_up()

// ── Trait declaration (Feature 30) ───────────────────────
trait Printable {
    print
    debug
}

// ── Destructuring (Feature 33) ────────────────────────────
let {name, score} = p
say "Name: {name}, Score: {score}"
Lambdas, Closures & Pattern Matching
Remox — Functional Programming
// ── Single-param lambda (Feature 39) ─────────────────────
let double = x => x * 2
say double(5)   // 10

// ── Multi-param lambda ────────────────────────────────────
let add = (a, b) => a + b
say add(3, 4)  // 7

// ── Closure — captures environment ───────────────────────
let base = 10
let adder = x => x + base   // base captured
say adder(5)    // 15

// ── Method chaining (Feature 25) ─────────────────────────
let cleaned = input.trim().lower().replace(" ", "_")

// ── Try / Catch (Feature 31) ──────────────────────────────
let data = try {
    risky_operation()
} catch err {
    say "Caught: {err}"
    null
}

// ── When (guard mode, no subject) ────────────────────────
when {
    x > 100 => say "High"
    x > 50  => say "Mid"
    else     => say "Low"
}

Malib — The Math Engine

Malib is a single-file Computer Algebra System (CAS) built from scratch inside Remox. It tokenizes, parses, and evaluates math expressions as strings — supporting variables, implicit multiplication, all trig functions, and equation solving via Newton-Raphson + bisection fallback.

Remox — Malib Examples
use Malib

// ── Expression Evaluation ─────────────────────────────────
say Malib.eval("2x + sin(pi/6)")   // x=0 by default

// ── Equation Solving ──────────────────────────────────────
say Malib.solve("x^2 - 5x + 6 = 0")  // [2, 3]
say Malib.solve("2x + 8 = 0")          // x = -4

// ── Calculus (Numerical) ──────────────────────────────────
say Malib.derivative("x^2", 3)   // ≈ 6.0 (slope at x=3)
say Malib.integral("x^2", 0, 3)   // ≈ 9.0

// ── Number Theory ─────────────────────────────────────────
say Malib.isPrime(17)         // true
say Malib.factorize(360)      // [2,2,2,3,3,5]
say Malib.gcd(48, 18)          // 6
say Malib.fibonacci(10)        // 55

// ── Combinatorics ─────────────────────────────────────────
say Malib.nCr(10, 3)            // 120
say Malib.nPr(10, 3)            // 720

// ── Statistics ────────────────────────────────────────────
let data = [4, 8, 6, 5, 3, 2, 8, 9]
say Malib.mean(data)             // 5.625
say Malib.median(data)           // 5.5
say Malib.stdev(data)            // standard deviation

// ── Matrices ──────────────────────────────────────────────
let m1 = [[1,2],[3,4]]
let m2 = [[5,6],[7,8]]
say Malib.matMul(m1, m2)
say Malib.matDet(m1)             // -2
Phinolib — Real Physics Engine

Phinolib contains exact, closed-form physics formulae — not simulations, not approximations. Every function maps to a real physics equation. All physical constants are embedded with full precision.

Remox — Phinolib Examples
use Phinolib

// ── Physical Constants ────────────────────────────────────
say Phinolib.G        // 6.674e-11 (Gravitational constant)
say Phinolib.c        // 299,792,458 m/s (Speed of light)

// ── Kinematics ────────────────────────────────────────────
say Phinolib.displacement(10, 2, 5)   // s = v0*t + 0.5*a*t²
say Phinolib.velocity(0, 9.8, 3)     // v = v0 + a*t

// ── Projectile Motion ─────────────────────────────────────
say Phinolib.projectileRange(50, 45)     // R = v0²·sin(2θ)/g
say Phinolib.projectileMaxHeight(50, 45) // H = v0²·sin²θ/2g

// ── Gravity & Orbital Mechanics ───────────────────────────
say Phinolib.escapeVelocity(5.97e24, 6.37e6)  // Earth escape v
say Phinolib.orbitalPeriod(1.99e30, 1.5e11)   // Earth year (Kepler)
say Phinolib.schwarzschildRadius(1.99e30)     // Sun's black hole r

// ── Special Relativity ────────────────────────────────────
let v = 0.9 * Phinolib.c       // 90% speed of light
say Phinolib.lorentzFactor(v)   // γ = 1/√(1-v²/c²)
say Phinolib.timeDilation(1, v)  // t = γ·t₀

// ── Electromagnetism ──────────────────────────────────────
say Phinolib.coulombForce(1e-6, 2e-6, 0.1)  // F = k·q1·q2/r²
say Phinolib.faradayEMF(100, 0.05, 0.01)    // ε = -N·dΦ/dt

// ── Fluid Mechanics ───────────────────────────────────────
say Phinolib.reynoldsNumber(1000, 2, 0.05, 0.001) // Re
say Phinolib.bernoulliV2(5, 200000, 100000, 1000, 0, 2)

// ── Optics ────────────────────────────────────────────────
say Phinolib.snellsLaw(1.0, 30, 1.5)    // refraction angle
say Phinolib.thinLens(20, 30)          // image distance

Numrux — NumPy-Equivalent Array Engine
Remox — Array Computing
use Numrux

// ── Array Creation ────────────────────────────────────────
let a = Numrux.array([[1,2,3],[4,5,6]])
let z = Numrux.zeros([3, 3])
let eye = Numrux.eye(3)          // Identity matrix
let rng = Numrux.arange(0, 10, 2) // [0,2,4,6,8]

// ── Broadcasting Math (+, -, *, / work directly) ──────────
let b = Numrux.array([10, 20, 30])
say a + b     // element-wise, broadcasting
say a * 2     // scalar multiplication

// ── Reductions ────────────────────────────────────────────
say Numrux.sum(a)
say Numrux.mean(a)
say Numrux.std(a)

// ── Linear Algebra ────────────────────────────────────────
say Numrux.dot(a, b)
say Numrux.det(eye)
say Numrux.transpose(a)

// ── Shape Operations ──────────────────────────────────────
say Numrux.shape(a)    // [2, 3]
say Numrux.reshape(a, [3, 2])

UI DSL & Mastery

Remox includes a built-in UI description language — write HTML/CSS-level interfaces directly in Remox syntax, and the runtime generates a complete HTML file for you.

The UI Block System
Remox — UI DSL (Features 41–50)
// ── Basic UI Block ────────────────────────────────────────
ui HomePage {
    title: "My App"
    div {
        backgroundColor: "#1a1a2e"
        padding: "20px"

        text "Welcome to Remox!" {
            fontSize: "32px"
            color: "#gold"
        }

        button "Get Started" {
            onClick: "handleClick"
            backgroundColor: "#gold"
        }

        input {
            placeholder: "Enter your name..."
            type: "text"
        }
    }
}

// ── Layout System (Features 47-49) ───────────────────────
ui Dashboard {
    layout row {
        div { width: "200px"  // Sidebar }
        div { flex: "1"       // Main content }
    }

    layout grid {
        columns: 3
        gap: 16
        div { // Card 1 }
        div { // Card 2 }
        div { // Card 3 }
    }
}

// ── Reusable View Components (Feature 41b) ────────────────
view NavBar {
    div {
        backgroundColor: "#dark"
        text "Remox App"
    }
}

// ── Scoped CSS (Feature 50) ───────────────────────────────
style {
    .hero {
        background: "linear-gradient(135deg, #1a1208, #2d1f08)"
        color: "white"
        padding: "60px"
    }
    #title {
        fontSize: "48px"
        fontWeight: "bold"
    }
}
⚠️
CSS Key Naming

Remox identifiers can't contain hyphens, so camelCase automatically converts to kebab-case. backgroundColor becomes background-color, fontSize becomes font-size.
